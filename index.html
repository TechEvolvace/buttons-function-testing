Unexpected Result

  • When Input Area is left empty, apparently it displays that it is actually a number, instead of the expected "It is actually not a number".

How the User Input Processing Lead to this unexpected result?

  1. When a user enters an input into the input area, the .value command receives the value inside that input area as a string.

  2. The isNaN command first converts that string value into a number before checking if that is not a number and returns the appropriate boolean value.

  3. When the user leaves nothing in the input area, the empty "" string value is converted to a number of 0.

  4. This number 0 is then what isNaN checks to see if it's not a number, which returns false.

  5. Thus, the message "It is actually a number." gets shown despite not having any number in that input area!

How to Account for Edge case of User Leaving Behind an Empty Input

To prevent this, consider having the input area converts from type "text" to type "number" instead!

How to Target a Specific Element in JavaScript:

There are 2 different ways to target and get a specific element:

  1. Use document.getElementbyId("name of element with a specific id value") to get a specific element that has that specific id value!

  2. Use document.querySelector("[tag class] name of element's specific value from that tag class")

    Specific tag class:

      . = targeting an element with a specific class name

      # = targeting an element with a specific id value

      * = universal selector that will target every element

      By default, no tag class will specify a specific tag element (such as body, p, ...)

        Examples:

        • Getting a specific tag element:

          document.querySelector("main")

        • Getting an element with the specific class name:

          document.querySelector(".button-design")

        • Getting an element with the specific id value:

          document.querySelector("#main-button")

About JavaScript's Handling of Child Nodes

  • Appending an element as a child node under another element will display that element as a separate element.

  • Without conditional statements to check if there is that child element you want to append under another element, then for each time the function that gets called that involves appending that child element, one child element will get appended each time. The final result is of many of the same child elements popping up on the website, which is mostly likely not your intended result!

  • For the following references to the conditional statement:

    [parent element] : the element to append the new element onto

    [new element] : the new element that was created and you wanted that element to append onto the parent element to get it displayed correctly in the website

  • The conditional statement to use to check if the parent element has a specific element as its appended child element is: [parent element].contains([new element])

    Ignore the brackets in the code above because [parent element] and [new element] are how I referenced the element you want to append a specfic element to and the specific element to append, respectively.

  • How to Use the Conditional Statement to Determine whether to either append a new element or not?

    • When the element is not initially shown on the website and needs to be appended to a specific element in the Document Object Model, you need to first check if the conditional statemnt [parent element].contains([new element]) is false before appending that new element.

    • When there's already an element initially shown on the website, it's already appended in the Document Object Model. If you need to just change the text content displayed in that element, just change the value assigned to that element's textContent property.
